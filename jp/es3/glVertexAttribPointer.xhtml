名前

glVertexAttribPointer — ジェネリック頂点属性データの配列を定義する

仕様

void glVertexAttribPointer( GLuint index,
  GLint size,
  GLenum type,
  GLboolean normalized,
  GLsizei stride,
  const GLvoid * pointer);
 
void glVertexAttribIPointer( GLuint index,
  GLint size,
  GLenum type,
  GLsizei stride,
  const GLvoid * pointer);
 

パラメーター

index

変更するジェネリック頂点属性のインデックスを指定する

size

ジェネリック頂点属性ごとのコンポーネントの数を指定する。1、2、3、4でなければならない。初期値は4。

type

配列のコンポーネントごとのデータ型を指定する。 使用できる定数は GL_BYTE、 GL_UNSIGNED_BYTE、 GL_SHORT、 GL_UNSIGNED_SHORT、 GL_INT、 GL_UNSIGNED_INTで、 追加で以下も使える。 GL_HALF_FLOAT、 GL_FLOAT、 GL_FIXED、 GL_INT_2_10_10_10_REV、 GL_UNSIGNED_INT_2_10_10_10_REVは、glVertexAttribPointerで使用できる。 初期値はGL_FLOAT。/p>

normalized

glVertexAttribPointerでは、 正規化されている固定小数点かどうかを指定し (GL_TRUE) または、アクセス済みの時に直接固定小数点に変換する or converted directly as fixed-point values (GL_FALSE) このパラメーターは、typeがGL_FIXEDの時に無視される。

stride

連続したジェネリック頂点属性の間のオフセットを指定する。 もしstrideが0なら, ジェネリック頂点属性は、しっかりとパックされた配列の中にあります。

pointer

配列の最初のジェネリック頂点属性へのポインタを指定する、 もし、0でないバッファが現在GL_ARRAY_BUFFERへバウンドされているなら、 そのバッファのデータストアの配列へのオフセットを指定する。 初期値は0。

説明

glVertexAttribPointerとglVertexAttribIPointer は、indexのジェネリック頂点属性の配列の位置と日付のフォーマットをレンダリングのために特定する。 属性ごとのコンポーネントの数をsizeに指定し、 1、2、3、4である必要がある。 コンポーネントごとのデータ型typeに指定し、一つの属性から次への飛ばすバイト数strideに指定し、 一つの配列への頂点と属性をパックしたり、個別の配列を保存したりできるようにする。

glVertexAttribPointerでは、もしnormalizedがGL_TRUEに設定されていれば、 整数形式に保存された値が範囲[-1,1](符号付き)かアクセス済みで浮動小数点の時に[0, 1](符号なし)にマッピングされること示す。 それ以外は値が直接正規化せずにfloatに変換されます。

glVertexAttribIPointerでは、整数型がGL_BYTEの時のみ, GL_UNSIGNED_BYTE、GL_SHORT、GL_UNSIGNED_SHORT、 GL_INT、GL_UNSIGNED_INTが受け入れられる。 値はいつも整数値のままである。

もし0でない名前つきバッファオブジェクトがGL_ARRAY_BUFFERターゲットへバウンドされたら (glBindBufferをみてください)、 pointerは、バッファオブジェクトのデータストアへのバイトオフセット扱われ、 バインディング(GL_ARRAY_BUFFER_BINDING)しているバッファオブジェクトは、ジェネリック頂点属性配列の 状態(GL_VERTEX_ATTRIB_ARRAY_BUFFER_BINDING)としてindex用に保存される。

(0のGL_ARRAY_BUFFERターゲットへのバインディングをされた)クライアント頂点配列は、 接続詞と一緒に0の名前付き頂点属性オブジェクトのなかでのみ有効だ。 これはOpenGL ES 2.0で後方互換性を提供する。

ジェネリック頂点属性が指定された時に、 size、type, normalized、 stride、そして pointer が頂点配列の状態として保存され、追加で正しい頂点配列バッファオブジェクトにバインディングされる。

ジェネリック頂点属性を有効化または無効化するために、 glEnableVertexAttribArrayや glDisableVertexAttribArrayをindexと一緒に呼び出す。 有効ならば、 glDrawArrays、 glDrawArraysInstanced、glDrawElements、 glDrawElementsIntanced、またはglDrawRangeElements が呼び出された時に ジェネリック頂点属性は使用されている。

ノート

Each generic vertex attribute array is initially disabled and isn't accessed when glDrawElements, glDrawRangeElements, glDrawArrays, glDrawArraysInstanced, or glDrawElementsInstanced is called.

エラー

GL_INVALID_VALUE is generated if index is greater than or equal to GL_MAX_VERTEX_ATTRIBS.

GL_INVALID_VALUE is generated if size is not 1, 2, 3 or 4.

GL_INVALID_ENUM is generated if type is not an accepted value.

GL_INVALID_VALUE is generated if stride is negative.

GL_INVALID_OPERATION is generated if type is GL_INT_2_10_10_10_REV or GL_UNSIGNED_INT_2_10_10_10_REV and size is not 4.

GL_INVALID_OPERATION is generated a non-zero vertex array object is bound, zero is bound to the GL_ARRAY_BUFFER buffer object binding point and the pointer argument is not NULL.
